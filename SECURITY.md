# Security Policy

Security is a top priority for ClientXCMS. We take all security vulnerabilities seriously and appreciate your efforts to responsibly disclose your findings.

## Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| 1.x     | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within ClientXCMS, please follow these guidelines:

### How to Report

1. **Email**: Send a detailed report to [security@clientxcms.com](mailto:security@clientxcms.com)
2. **GitHub**: Use the "Report a vulnerability" button in the Security tab

### What to Include

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any proof-of-concept code (if applicable)
- Your contact information for follow-up

### What to Expect

- **Acknowledgment**: We will acknowledge receipt of your report within 48 hours
- **Assessment**: Our security team will assess the vulnerability and its impact
- **Updates**: We will keep you informed about our progress
- **Resolution**: Once resolved, we will notify you and credit you (if desired)

### Responsible Disclosure

- **Do NOT** disclose the vulnerability publicly until we have addressed it
- **Do NOT** use public channels (issues, discussions, forums) to report security vulnerabilities
- Allow us reasonable time to fix the issue before any disclosure

### Recognition

We value the security research community. Depending on the severity and impact of the vulnerability, we may offer:

- Public acknowledgment in our release notes
- A place in our Security Hall of Fame

# Politique de Sécurité

La sécurité est une priorité absolue pour ClientXCMS. Nous prenons toutes les vulnérabilités de sécurité au sérieux et apprécions vos efforts pour divulguer vos découvertes de manière responsable.

## Versions Supportées

Nous fournissons des mises à jour de sécurité pour les versions suivantes :

| Version | Supportée          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| 1.x     | :x:                |

## Signalement d'une Vulnérabilité

Si vous découvrez une vulnérabilité de sécurité dans ClientXCMS, veuillez suivre ces directives :

### Comment Signaler

1. **Email** : Envoyez un rapport détaillé à [security@clientxcms.com](mailto:security@clientxcms.com)
2. **GitHub** : Utilisez le bouton "Report a vulnerability" dans l'onglet Security

### Informations à Fournir

- Une description claire de la vulnérabilité
- Les étapes pour reproduire le problème
- L'impact potentiel de la vulnérabilité
- Tout code de preuve de concept (si applicable)
- Vos coordonnées pour un suivi

### À Quoi S'attendre

- **Accusé de réception** : Nous accuserons réception de votre signalement sous 48 heures
- **Évaluation** : Notre équipe de sécurité évaluera la vulnérabilité et son impact
- **Mises à jour** : Nous vous tiendrons informé de notre progression
- **Résolution** : Une fois résolue, nous vous notifierons et vous créditerons (si souhaité)

### Divulgation Responsable

- **NE divulguez PAS** la vulnérabilité publiquement avant que nous l'ayons corrigée
- **N'utilisez PAS** les canaux publics (issues, discussions, forums) pour signaler des vulnérabilités
- Accordez-nous un délai raisonnable pour corriger le problème avant toute divulgation

### Reconnaissance

Nous valorisons la communauté de recherche en sécurité. Selon la gravité et l'impact de la vulnérabilité, nous pouvons offrir :

- Une reconnaissance publique dans nos notes de version
- Une place dans notre Hall of Fame de la Sécurité
